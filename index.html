GeoJSON Usage Tutorial

This tutorial shows how to prepare and use a GeoJSON file to define soil/terrain regions for Metro Manila (or any region).

1. Prepare a GeoJSON file: Your GeoJSON should have Feature objects with a properties.soil_type field whose values match keys in the Mapping JSON.

Example GeoJSON snippet:

{
  "type": "FeatureCollection",
  "features": [
    {
      "type": "Feature",
      "properties": { "soil_type": "Coastal Lowland" },
      "geometry": {
        "type": "Polygon",
        "coordinates": [
          [
            [120.95,14.55], [120.98,14.55], [120.98,14.52], [120.95,14.52], [120.95,14.55]
          ]
        ]
      }
    },
    {
      "type": "Feature",
      "properties": { "soil_type": "Marikina Valley" },
      "geometry": {
        "type": "Polygon",
        "coordinates": [
          [
            [121.05,14.65], [121.10,14.65], [121.10,14.60], [121.05,14.60], [121.05,14.65]
          ]
        ]
      }
    }
    // ... more features ...
  ]
}

2. Mapping JSON: Provide a JSON object mapping each soil_type string to the corresponding medium ID. For example:

{
  "Water": 2,
  "Coastal Lowland": 5,
  "Central Plateau": 6,
  "Marikina Valley": 7,
  "Novaliches Loam": 8,
  "San Luis Clay": 9,
  "Soft Soil": 0,
  "Sand": 3,
  "Rock": 4
}

3. Set bounding box: Enter latMin, latMax, lonMin, lonMax covering your GeoJSON region (e.g., Metro Manila approx 14.4–14.8 lat, 120.9–121.2 lon). The grid will map each cell’s center to a lat/lon via linear interpolation.

4. Load GeoJSON: Click “Load GeoJSON” and ensure status says “GeoJSON loaded.”

5. Apply Terrain: Click “Apply Terrain” to fill the grid according to your GeoJSON and mapping.

If any cell falls outside all features, it defaults to Soft Soil (ID 0). You can adjust default behavior in code if needed.
